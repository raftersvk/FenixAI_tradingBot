**Date** : 2025-04-19  
**Projet** : FenixAI Trading Bot  
**Objectif** : Unifier tous les systèmes de configuration sur `src/config/unified_loader.py` + `config/fenix.yaml` et nettoyer les variables d'environnement obsolètes du `.env`.

## 📊 ÉTAT ACTUEL — Problèmes Identifiés

### 1.1 Systèmes de Configuration Parallèles (4 systèmes)

| Source | Type | Chargé par | Utilisé par | Problème |
|--------|------|------------|-------------|----------|
| `config/fenix.yaml` | YAML | `unified_loader.py` | **Presque personne** | Ignoré par TradingEngine |
| `config/config.yaml` | YAML | `config_loader.py` → `APP_CONFIG` | API server, Engine (partiel) | Ancien système, doublons |
| `config/settings.py` | Python dataclass | Aucun (module arm) | Double usage | Incohérent |
| `src/config/system_config.yaml` | YAML | Inconnu | ? | Obsolète, inutilisé |
| Variables d'env (`os.getenv`) | Shell | Partout | Tous modules | Éparpillé, non documenté |

### 1.2Paramètre Critique `min_klines_to_start`

**Valeur dans `config/fenix.yaml`** : `500`  
**Valeur utilisée par `TradingEngine`** : `20` (depuis `os.getenv("FENIX_MIN_KLINES_TO_START", "20")`)  
**Cause** : `TradingEngine.__init__` (ligne 149) ne lit **aucun** fichier YAML, seulement une variable d'environnement.

### 1.3 Fichiers Obsolètes dans le repo

- `config/settings.py` (348 lignes) — pas importé nulle part ? (à vérifier)
- `config/system_config.yaml` (390 lignes) — inutilisé
- `config/config.yaml` (252 lignes) — ancien système, doublonne avec `fenix.yaml`
- `src/config/trading_constants.py` (258 lignes) — charge depuis JSON, pas de fichier trouvé

```python
def __init__(
    self,
    symbol: str | None = None,
    timeframe: str | None = None,
    use_testnet: bool | None = None,
    paper_trading: bool | None = None,
    enable_visual_agent: bool | None = None,
    enable_sentiment_agent: bool | None = None,
    allow_live_trading: bool = False,
):
    # Config unifiée
    if _UNIFIED_CONFIG_AVAILABLE:
        config = get_config()
        self.symbol = (symbol or config.trading.symbol).upper()
        self.timeframe = timeframe or config.trading.timeframe
        self._min_klines_to_start = config.trading.min_klines_to_start
        
        # use_testnet: priorité args > config
        self.use_testnet = use_testnet if use_testnet is not None else config.binance.testnet
        
        # paper_trading: priorité args > config
        if paper_trading is not None:
            self.paper_trading = paper_trading
        else:
            # Par défaut, paper = true si testnet ou pas de live trading explicite
            self.paper_trading = not allow_live_trading or self.use_testnet
        
        # Agents
        self.enable_visual = enable_visual_agent if enable_visual_agent is not None else config.agents.enable_visual
        self.enable_sentiment = enable_sentiment_agent if enable_sentiment_agent is not None else config.agents.enable_sentiment
    else:
        # Fallback vers ancienne logique
        self.symbol = (symbol or "BTCUSDT").upper()
        self.timeframe = timeframe or "15m"
        self.use_testnet = use_testnet if use_testnet is not None else False
        self.paper_trading = paper_trading if paper_trading is not None else True
        self.enable_visual = enable_visual_agent if enable_visual_agent is not None else True
        self.enable_sentiment = enable_sentiment_agent if enable_sentiment_agent is not None else True
        self._min_klines_to_start = int(os.getenv("FENIX_MIN_KLINES_TO_START", "20"))

    self.allow_live_trading = allow_live_trading
    # ...
```

## ⚠️ RISQUES ET CONTINGENCES

| Risque | Impact | Contournement |
|--------|--------|--------------|
| `get_config()` échoue (fichier manquant) | TradingEngine crash | Fallback vers defaults + warning |
| `fenix.yaml` malformé (YAML invalide) | App ne démarre pas | Validation YAML avant commit |
| `unified_loader` pas encore prêt pour toutes les sections | Champs manquants | Ajouter progressivement les sections `api`, etc. |
| Tests existants cassés | CI échoue | Mocker `get_config()` dans tous les tests |
| Variables `.env` supprimées mais encore utilisées somewhere | Crash au démarrage | Grep complet avant suppression: `grep -r "VAR_NAME" src/` |
| Migration Docker incomplète | Conteneur échoue | Tester en local avant docker-compose up |
